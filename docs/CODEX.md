# SmartAccounting – Engineering Codex

## Principles

- Type-safe by default
- No silent failures
- Deterministic financial logic
- Explicit error handling
- CI before merge

## Rules

- ESLint must pass
- Tests required for accounting logic
- No direct DB writes outside services
- No console.log in production
- Docker must build cleanly

## Phases

- PLAN → IMPLEMENT → VALIDATE → COMMIT
